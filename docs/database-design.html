4 Database Design

4.1 Database Architecture

4.1.1 Database Technology Stack

The system uses SQLite with Prisma ORM for type-safe database operations. This combination provides an excellent balance of simplicity, performance, and developer productivity:

🗄️ SQLite Database

  • Serverless, file-based database for easy deployment
  • ACID compliance for data integrity
  • Zero configuration and maintenance
  • Excellent performance for read-heavy operations

🔮 Prisma ORM

  • Type-safe database access with auto-generated client
  • Declarative schema definition with migrations
  • Excellent developer experience with Prisma Studio
  • Built-in query optimization and caching

🛠️ Development Tools

  • Prisma Studio for visual database management
  • Automatic schema migrations
  • Database seeding for development
  • Query introspection and debugging

4.1.2 Design Principles

The database design follows established principles to ensure data integrity, performance, and maintainability:

  • Normalization: 3NF compliance to eliminate data redundancy and ensure data integrity
  • Referential Integrity: Foreign key constraints with cascading rules to maintain consistent relationships
  • Performance Optimization: Strategic indexing for frequently queried columns and join operations
  • Scalability: Design supporting future data growth and potential migration to larger databases
  • Security: Protection of sensitive user and operational data through proper access controls
  • Auditability: Comprehensive logging of system activities for compliance and dispute resolution

4.2 Entity Relationship Diagram Explanation

4.2.1 Core Entities

The system comprises eight primary entities with well-defined relationships supporting the complete waste management workflow:

👤 User

Central entity for all system users (Citizens, Admins, Drivers)

  • Authentication credentials and profile information
  • Role-based permissions and access control
  • Relationship to all other entities through user actions

📝 GarbageReport

Citizen pickup requests with complete lifecycle tracking

  • Photo evidence and GPS location data
  • Status progression through approval workflow
  • Assignment to workers and task creation

📋 Task

Worker assignments linking reports to field operations

  • Scheduled collection times and locations
  • Status tracking and completion verification
  • Proof of work documentation

👷 Worker

Extended driver profiles for workforce management

  • Performance metrics and workload tracking
  • Availability and capacity management
  • Specialized worker attributes and skills

💬 Feedback

Citizen feedback and admin responses with AI analysis

  • Categorized feedback types and priorities
  • AI-powered content moderation
  • Resolution tracking and admin responses

⚠️ Issue

Dispute resolution and service issue tracking

  • Service quality complaints and disputes
  • Escalation workflows and resolution tracking
  • Link to specific reports and workers

🔔 Notification

System communications and user alerts

  • Role-based notification filtering
  • Priority levels and delivery tracking
  • Read/unread status management

📋 AuditLog

Comprehensive activity tracking for security and compliance

  • All system actions with user context
  • Entity change history and values
  • Security event monitoring and alerting

4.2.2 Relationship Types

  • One-to-Many: User to GarbageReports, User to Tasks, User to Notifications
  • One-to-One: User to Worker profile, GarbageReport to primary Task
  • Many-to-Many: Resolved through junction tables for complex relationships
  • Recursive: Self-referencing relationships for hierarchical data

4.3 Table Structures

4.3.1 User Table

Purpose: Stores authentication credentials and profile information for all system users

CREATE TABLE Users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL, -- Admin, Worker, Citizen
    profileImage VARCHAR(255),
    createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    updatedAt DATETIME DEFAULT CURRENT_TIMESTAMP
);

Relationships:

  • One-to-many with GarbageRequest (citizen requests)
  • One-to-many with Task (driver assignments)
  • One-to-many with Feedback (citizen feedback)
  • One-to-many with Issue (user issues)
  • One-to-one with Worker (driver profile)

4.3.2 GarbageReport Table

Purpose: Manages citizen garbage pickup requests with complete lifecycle tracking

CREATE TABLE GarbageReports (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    imagePath VARCHAR(255) NOT NULL,
    latitude REAL NOT NULL,
    longitude REAL NOT NULL,
    status VARCHAR(50) DEFAULT 'REPORTED',
    citizenId INTEGER NOT NULL,
    assignedWorkerId INTEGER,
    preferredDate DATETIME,
    address VARCHAR(255),
    notes TEXT,
    adminNotes TEXT,
    statusHistory TEXT, -- JSON string
    createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    updatedAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (citizenId) REFERENCES Users(id),
    FOREIGN KEY (assignedWorkerId) REFERENCES Users(id)
);

Relationships:

  • Many-to-one with User (citizen and driver)
  • One-to-one with Task (assignment)
  • One-to-many with Issue (related disputes)

4.3.3 Task Table

Purpose: Tracks driver assignments and task execution with proof of work

CREATE TABLE Tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    garbageReportId INTEGER,
    latitude REAL NOT NULL,
    longitude REAL NOT NULL,
    scheduledTime DATETIME NOT NULL,
    status VARCHAR(50) DEFAULT 'ASSIGNED',
    workerId INTEGER NOT NULL,
    statusHistory TEXT, -- JSON string
    completedAt DATETIME,
    acceptedAt DATETIME,
    unableReason TEXT,
    proofImage VARCHAR(255),
    createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    updatedAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (garbageReportId) REFERENCES GarbageReports(id),
    FOREIGN KEY (workerId) REFERENCES Users(id)
);

Relationships:

  • Many-to-one with GarbageReport
  • Many-to-one with User (driver)

4.3.4 Worker Table

Purpose: Extended profile information for driver workforce management

CREATE TABLE Workers (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    userId INTEGER UNIQUE NOT NULL,
    currentWorkload INTEGER DEFAULT 0,
    maxTasks INTEGER DEFAULT 10,
    performanceRating REAL DEFAULT 0,
    totalTasksCompleted INTEGER DEFAULT 0,
    createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    updatedAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (userId) REFERENCES Users(id)
);

Relationships:

  • One-to-one with User
  • One-to-many with Task
  • One-to-many with Feedback

4.3.5 Feedback Table

Purpose: Manages citizen feedback, complaints, and administrator responses

CREATE TABLE Feedback (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    type VARCHAR(50) NOT NULL, -- COMPLAINT, SUGGESTION, ISSUE, COMPLIMENT
    title VARCHAR(255) NOT NULL,
    description TEXT NOT NULL,
    category VARCHAR(50) NOT NULL, -- SERVICE, WORKER, SYSTEM, OTHER
    priority VARCHAR(20) DEFAULT 'MEDIUM', -- LOW, MEDIUM, HIGH, URGENT
    status VARCHAR(20) DEFAULT 'OPEN', -- OPEN, IN_PROGRESS, RESOLVED, REJECTED
    citizenId INTEGER NOT NULL,
    adminId INTEGER,
    workerId INTEGER,
    adminReply TEXT,
    adminNotes TEXT,
    replied BOOLEAN DEFAULT FALSE,
    aiAnalysis TEXT, -- JSON string of AI content analysis
    isFlagged BOOLEAN DEFAULT FALSE,
    flaggedAt DATETIME,
    createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    updatedAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (citizenId) REFERENCES Users(id),
    FOREIGN KEY (adminId) REFERENCES Users(id),
    FOREIGN KEY (workerId) REFERENCES Workers(id)
);

Relationships:

  • Many-to-one with User (citizen and admin)
  • Many-to-one with Worker

4.3.6 Issue Table

Purpose: Handles dispute resolution, service issues, and quality assurance

CREATE TABLE Issues (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    type VARCHAR(50) NOT NULL, -- DISPUTE, SERVICE_ISSUE, TECHNICAL
    title VARCHAR(255) NOT NULL,
    description TEXT NOT NULL,
    category VARCHAR(50) NOT NULL, -- REPORT_REJECTED, WORKER_CONDUCT, SYSTEM_ERROR, OTHER
    priority VARCHAR(20) DEFAULT 'MEDIUM', -- LOW, MEDIUM, HIGH, URGENT
    status VARCHAR(20) DEFAULT 'PENDING', -- PENDING, IN_REVIEW, RESOLVED, REJECTED
    citizenId INTEGER NOT NULL,
    workerId INTEGER,
    adminId INTEGER,
    garbageReportId INTEGER,
    resolution TEXT,
    replied BOOLEAN DEFAULT FALSE,
    createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    updatedAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (citizenId) REFERENCES Users(id),
    FOREIGN KEY (workerId) REFERENCES Users(id),
    FOREIGN KEY (adminId) REFERENCES Users(id),
    FOREIGN KEY (garbageReportId) REFERENCES GarbageReports(id)
);

Relationships:

  • Many-to-one with User (citizen, driver, admin)
  • Many-to-one with GarbageReport

4.3.7 Notification Table

Purpose: Manages system notifications and user communications

CREATE TABLE Notifications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title VARCHAR(255) NOT NULL,
    message TEXT NOT NULL,
    type VARCHAR(20) NOT NULL, -- INFO, SUCCESS, WARNING, ERROR
    category VARCHAR(20) NOT NULL, -- REPORT, TASK, FEEDBACK, ISSUE, SYSTEM
    isRead BOOLEAN DEFAULT FALSE,
    userId INTEGER NOT NULL,
    relatedId INTEGER, -- ID of related entity
    priority VARCHAR(10) DEFAULT 'NORMAL', -- LOW, NORMAL, HIGH, URGENT
    expiresAt DATETIME,
    createdAt DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (userId) REFERENCES Users(id)
);

Relationships:

  • Many-to-one with User

4.3.8 AuditLog Table

Purpose: Comprehensive audit trail for security and compliance

CREATE TABLE AuditLogs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    action VARCHAR(50) NOT NULL, -- CREATE, UPDATE, DELETE, LOGIN, LOGOUT, ASSIGN, COMPLETE
    entity VARCHAR(50) NOT NULL, -- USER, REPORT, TASK, FEEDBACK, ISSUE
    entityId INTEGER,
    oldValues TEXT, -- JSON string of old values
    newValues TEXT, -- JSON string of new values
    userId INTEGER NOT NULL,
    userAgent TEXT,
    ipAddress VARCHAR(45),
    sessionId VARCHAR(255),
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (userId) REFERENCES Users(id)
);

Relationships:

  • Many-to-one with User

4.4 Notification System Design

4.4.1 Notification Architecture

The notification system provides real-time communication between all system stakeholders through a centralized notification management framework that ensures timely and relevant information delivery.

4.4.2 Notification Types

📈 Status Change Alerts

  • Request status updates (submitted, approved, assigned, completed)
  • Task progress notifications (accepted, in-progress, completed)
  • Feedback and issue status changes
  • System maintenance and downtime notifications

📋 Task Assignment Notifications

  • New task assignments to workers
  • Task reassignment notifications
  • Deadline reminders and urgent tasks
  • Workload management alerts

✅ Completion Notifications

  • Service completion confirmations to citizens
  • Proof of work notifications
  • Quality assurance completion alerts
  • Performance milestone achievements

💬 Feedback Alerts

  • New feedback submissions to administrators
  • Admin response notifications to citizens
  • AI moderation alerts for flagged content
  • Resolution notifications for closed feedback

4.4.3 Notification Flow

Event Trigger → Notification Generation → User Filtering → Database Storage → Real-time Delivery

4.4.4 Notification Features

  • Role-Based Filtering: Notifications filtered by user role and relevance to reduce noise
  • Read/Unread Tracking: Comprehensive notification status management with read receipts
  • Priority Levels: Critical, high, medium, low priority notifications with appropriate handling
  • Batch Processing: Efficient notification delivery for multiple users and bulk operations
  • History Management: Complete notification history with search and filtering capabilities
  • Expiration Management: Automatic cleanup of expired notifications to maintain performance

4.5 Audit Logging System

4.5.1 Audit System Architecture

The audit logging system provides comprehensive activity tracking for security, compliance, and dispute resolution. Every significant action in the system is automatically logged with detailed context, ensuring complete transparency and accountability.

4.5.2 Logged Activities

👤 User Actions

  • Login, logout, profile changes, password updates
  • Account creation and deletion
  • Permission changes and role modifications
  • Preference and setting updates

📝 Report Operations

  • Report creation, modification, deletion
  • Approval and rejection actions
  • Status changes and transitions
  • Assignment and reassignment operations

📋 Task Management

  • Task creation and assignment
  • Status updates and progress changes
  • Completion verification and proof upload
  • Escalation and reassignment actions

🔐 Security Events

  • Failed login attempts and account lockouts
  • Permission violations and access denials
  • Suspicious activities and anomaly detection
  • Security configuration changes

4.5.3 Audit Data Structure

  • Action Type: Categorized operation (CREATE, UPDATE, DELETE, LOGIN, LOGOUT, ASSIGN, COMPLETE)
  • Entity Information: Affected database entity and ID for precise tracking
  • Change Tracking: Before/after values for update operations in JSON format
  • User Context: User ID, IP address, user agent, and session information
  • Timestamp: Precise event timing with timezone support for chronological analysis

4.5.4 Audit System Benefits

  • Transparency: Complete visibility into all system operations and user activities
  • Accountability: Clear responsibility tracking for all actions and decisions
  • Security: Early detection of suspicious activities and potential security breaches
  • Compliance: Support for regulatory requirements and audit obligations
  • Dispute Resolution: Historical evidence for issue investigation and conflict resolution
  • Performance Analysis: Data for system usage patterns and optimization opportunities

4.6 Analytics System

4.6.1 Analytics Dashboard Architecture

The analytics system provides comprehensive insights into system performance, worker productivity, and service quality through interactive dashboards and detailed reports that enable data-driven decision making.

4.6.2 Key Performance Indicators

📊 Request Metrics

  • Total requests and submission trends
  • Approval rate and rejection analysis
  • Average resolution time and processing efficiency
  • Request distribution by location and time

👷 Worker Performance

  • Tasks completed per worker and productivity rates
  • Average completion time and efficiency metrics
  • Photo upload compliance and quality scores
  • Customer satisfaction ratings and feedback

⚡ System Activity

  • User engagement patterns and peak usage times
  • System response times and performance metrics
  • Error rates and system reliability statistics
  • Resource utilization and capacity planning

🎯 Service Quality

  • Citizen satisfaction scores and trends
  • Feedback resolution rates and response times
  • Issue tracking and dispute resolution metrics
  • Service level agreement compliance

4.6.3 Analytics Features

  • Real-Time Dashboards: Live data visualization with auto-refresh and real-time updates
  • Historical Trends: Time-series analysis for performance tracking and trend identification
  • Comparative Analysis: Worker and period-over-period comparisons with benchmarking
  • Export Capabilities: PDF and Excel export for reports and compliance documentation
  • Custom Reports: User-defined report parameters and filters for specialized analysis
  • Predictive Analytics: Forecasting capabilities for resource planning and demand prediction

4.6.4 Data Visualization

  • Charts and Graphs: Bar charts, line graphs, pie charts for data representation
  • Heat Maps: Geographic visualization of request density and service coverage
  • Performance Metrics: KPI cards with trend indicators and target comparisons
  • Interactive Filters: Dynamic data filtering and drill-down capabilities
  • Custom Visualizations: Specialized charts for waste management metrics

🎯 Database Performance Optimization

Several optimization strategies ensure excellent database performance:

  • Strategic Indexing: Primary and secondary indexes on frequently queried columns
  • Query Optimization: Efficient SQL queries with proper joins and filtering
  • Connection Pooling: Database connection management for concurrent access
  • Caching Strategies: Application-level caching for frequently accessed data
  • Regular Maintenance: Database optimization and statistics updates
